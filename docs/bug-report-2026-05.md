## カテゴリG: 新曲追加時の連鎖バグ（追記 2026-05-17）

新曲をホームページに追加するたびに、以下のバグが連鎖して発生している。

スクリプト実行後のチェックリストがなく、抜け漏れが常態化している。

## バグ頻発パターンと対策

### パターン1: 音声プツプツの再発サイクル
状況: 新機能追加（再生関連）→ プツプツ再発 → fixコミット → 別の新機能 → 再発
対策: 再生系コードを変更するたびに「プツプツチェックリスト」を必ず実行する。

### パターン2: ジャンルデータの4ファイル非同期
状況: スクリプトが songs.json を更新 → genres.json / by-genre/ / index.json が追随しない
対策: ジャンル変更時は必ず4ファイルを一括更新するスクリプトを通す。

### パターン3: SW キャッシュバージョン更新漏れ
状況: index.html を更新 → sw.js のバージョン番号を上げ忘れ → 古い画面が配信される
対策: index.html を変更したら sw.js のキャッシュバージョンも必ず上げる。

### パターン4: クリーンアップによる意図しない削除
状況: 孤立曲の一括削除 → 別ジャンルへ移動中の曲も巻き込まれる → ジャンル消滅
対策: 削除前に「この曲IDは genres.json / by-genre/*.json に存在するか」を確認する。

### パターン5: PRブランチ間の修正競合
状況: ブランチAで修正 → ブランチBで別修正 → B を main にマージ → A の修正が消える
対策: 修正PRを作る前に main から最新を pull し、ブランチを常に最新 main ベースにする。
